# 为什么选择 Burn？

既然 PyTorch、TensorFlow 和其他框架已经存在，为什么还要费力从头开始创建一个全新的深度学习框架呢？剧透提醒：Burn 不仅仅是 Rust 中的 PyTorch 或 TensorFlow 的复制版。它代表了一种新颖的方法，重点在于在正确的领域做出正确的妥协，以实现卓越的灵活性、高性能和无缝的开发者体验。Burn 不是一个专门用于单一类型应用的框架，而是设计为一个多功能框架，适用于广泛的研究和生产用途。Burn 的设计基础围绕三个关键用户群体：

**机器学习研究人员**需要高效的工具来构建和执行实验。对他们来说，快速迭代想法并设计可测试的实验以帮助他们发现新成果至关重要。框架应该促进前沿研究的快速实现，同时确保测试时的快速执行。

**机器学习工程师**是另一个需要牢记的重要群体。他们的关注点较少放在快速实现上，而更多地放在建立稳健性、无缝部署和成本效益的操作上。他们寻求可靠、经济的模型，能够在不过度消耗资源的情况下实现目标。整个机器学习工作流程——从训练到推理——必须尽可能高效，并且具有最少的不可预测行为。

**底层软件工程师**（与硬件供应商合作）希望他们的处理单元尽可能快地运行模型以获得竞争优势。这项工作涉及利用硬件特定的功能，如 Nvidia 的 Tensor Core。由于他们主要在系统层面工作，他们希望对计算执行方式拥有绝对控制权。

Burn 的目标是满足所有这些用户群体的需求！
